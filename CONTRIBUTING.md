# How to contribute to `scoreboard-rs`

## Community

Tolerance is a social contract, not a paradox. Be a kind, respectful person or you will be removed from this community.

## Setting up your development environment

1. Install the [Rust toolchain](https://rustup.rs/)
2. Download the `scoreboard-rs` repository
3. `cargo build` or `cargo run`

## PRs

Each PR will require two approving reviews to merge, regardless of who opens it. Those reviews can come from **anyone**,
and this will mark them as "ready". Maintainers will go through the list of "ready" PRs each week and also review
them, and either kick them back to the owner/reviewers with constructive feedback or merge them.

If you're a technical person, great! Please feel free to contribute how you see fit!

If you're not a technical person? That's fine too! Feel free to review code, and if you have questions ask them. PR
owners should be able to answer any questions you have. Asking and answering questions will ultimately help with the
posterity of understanding for the codebase and also help folks become better reviewers, so please ask as many questions
as you need!

# LLM and Generative AI Policy

### LLM and Generative AI Features

`scoreboard-rs` contains no LLM or generative AI features. `scoreboard-rs` commits to never including LLM or generative
AI
features.

### LLM and Generative AI Development

`scoreboard-rs` is not developed with LLMs or generative AI.

We acknowledge that contributors may have a variety of methodologies using generative AI assistance, such as generative
AI code reviews. We ask that all contributions to `scoreboard-rs` be substantially authored, understood, and discussed
by
humans.
We do not accept “vibe-coded” contributions, including code or assets that are created wholly or substantially by
generative AI.

**note: LLM and Generative AI Policy copied
from [Stave's LLM and Generative AI Policy](https://docs.stave.app/llm-generative-ai-policy.html).*
